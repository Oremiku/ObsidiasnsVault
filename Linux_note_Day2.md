# ディレクトリ構造について
![[Pasted image 20260706232359.png]]

### Windowsとの違い
![[Pasted image 20260706232543.png]]

- **Windows**のディレクトリ構造
	<mark>ドライブをルート</mark>としてツリーが作成される。
	2つのドライブがあった場合、2つのディレクトリツリーが作成される。

- **Linux**のディレクトリ構造
	「/」(<mark>ルートディレクトリ)をルート</mark>としてツリーが作成される。
	つまり、いかなる場合でもツリーは一つ。
	複数ディスクを用いる場合、ディスクをディレクトリとして使う
	**マウント**という機能がある。

### 主要なディレクトリ

- **/bin**
	- 一般ユーザーおよび管理ユーザーの両方が使用するコマンドの実行ファイルを格納
	- 特に、Linuxの動作に最低限必要な重要度の高いコマンドの実行ファイルを格納

- **/dev**
	- デバイスファイル(ハードウェアをファイルとして扱えるようにしたファイル)を格納

- **/etc**
	- 設定ファイルを格納
	- Linuxで動作するアプリケーションの設定ファイルや、Linux自体の設定に関わるファイルが置かれている。

- **/home** 🌟
	- ホームディレクトリ(ユーザーごとに割り当てられる個人用ディレクトリ)
	- 基本的にこのディレクトリ内で<mark>自由にファイルやディレクトリを作成</mark>できる。

- **/sbin**
	- 管理者ユーザー向けのコマンドの実行ファイルを格納

- **/tmp**
	- 一時的なファイルを格納
	- アプリケーションの実行中に結果を一時的にファイルとして保存する場合に使用

- **/usr**
	- 各種アプリケーションと、それに付随するファイルを格納

- **/var**
	- 変化するデータを格納
	- アプリケーションを動作する上で作成されたログなどを格納

# コマンド

### cd

#### コマンド
```markdown
cd [オプション] [ディレクトリ]
```

#### 概要
ディレクトリを移動するときに使用する。

#### ディレクトリの指定
| ディレクトリ     | 意味                  | 指定方法 |
| ---------- | ------------------- | ---- |
| カレントディレクトリ | 現在自分がいるディレクトリ       | 「.」  |
| 親ディレクトリ    | 一つ上の階層のディレクトリ       | 「..」 |
| ホームディレクトリ  | 「/home/ユーザー名」ディレクトリ | 「~」  |
| ルートディレクトリ  | 「/」ディレクトリ           | 「/」  |

### ls
#### コマンド
```markdown
ls [オプション] [ディレクトリ・ファイル名]
```

#### 具体例
```markdown
- ルート直下のディレクトリ・ファイルの一覧を表示
	ls /
	
- 複数のディレクトリを指定
	ls / /dev

```

#### 概要
ファイルやディレクトリの一覧を表示するコマンド。

#### パス名展開について
![[Pasted image 20260707000436.png]]
##### 例
カレントディレクトリに
- `index.html`
- `home.html`
があるとする。
以下のコマンドを実行する。
```
ls *.html
```
すると、内部的に以下のようになる。・
```
ls index.html home.html
```
このように、<mark>内部的に一致する全てのパス名へ変換される</mark>ため、
パス名展開という。
#### オプション
コマンドにオプションを与えることで、コマンドの動作を変えることができる。
![[Pasted image 20260707000803.png]]

![[Pasted image 20260707000859.png]]
長い名前が付いたオプション = ロングオプション

#### よく使うオプション
1. ファイルの詳細情報を表示
```
ls -l
```

2. 隠しファイルも含めたすべてのファイルを表示する
```
ls -a
```

3. ファイル種別を表示する
```
ls -F
```
↓

| 記号   | 種別        |
| ---- | --------- |
| /    | ディレクトリ    |
| @    | シンボリックリンク |
| \*   | 実行可能ファイル  |
| 記号無し | 通常のファイル   |

### mkdir

#### コマンド
```markdown
mkdir [オプション] <ディレクトリ名>
```

#### 概要
ディレクトリを作成することができるコマンド

#### よく使うオプション

- 深いディレクトリを一度に作成する
```markdown
mkdir -p ディレクトリ1/ディレクトリ2/ディレクトリ3....
```


### rmdir

#### コマンド
```
rmdir <ディレクトリ名>
```

#### 概要
<mark>空のディレクトリを削除する</mark>コマンド。
空でないディレクトリを削除しようとするとエラー。

## --helpオプション
コマンドの使い方を表示することが出来る
```
<コマンド> --help
```

例
```
ls --help
```

## manコマンド
--helpより詳しい使い方を表示することができる。
知らないコマンドは<mark>ググるよりまずmanコマンドを見る。</mark>
```
man <コマンド>
```

#### オプション
```
man -k <キーワード>
```
キーワードが含まれるコマンドの一覧を表示する
