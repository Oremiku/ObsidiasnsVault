# ファイルの中身を表示する

#### コマンド
```markdown
cat [オプション] <ファイル名>
```

#### 具体例
```markdown
- 基本
	cat  /etc/hosts

- 複数のファイルをまとめて表示することもできる
	cat /etc/hosts /etc/hosts
```

#### オプション
- 行番号を表示する
	`-n`


# ファイルの中身をスクロール表示する

#### コマンド
```
less [オプション] <ファイル名>
```

#### 具体例
```
less /etc/vimrc
```

#### 概要
ファイルの中身をスクロール表示するコマンド。
長いファイルの中身を見たいときに使う。

#### スクロールの操作
| コマンド | 内容         |
| ---- | ---------- |
| スペース | 一画面下にスクロール |
| b    | 一画面上にスクロール |
| j    | 一行下にスクロール  |
| k    | 一行上にスクロール  |
| q    | lessコマンド終了 |

| コマンド   | 内容         |
| ------ | ---------- |
| /<文字列> | 下方向に向かって検索 |
| ?<文字列> | 上方向に向かって検索 |
| n      | 次の検索結果に移動  |
| N      | 前の検索結果に移動  |

- バイナリファイル

# ファイルの作成

#### コマンド
```markdown
touch <新規ファイル名> <新規ファイル名2>...
```

#### 具体例
```markdown
touch README.md
```

#### 概要
ファイル名が存在していないときは空のファイルを作成。
存在しているときはタイムスタンプを更新。

つまり、<mark>間違えて存在するファイルを指定しても上書きしない</mark>為、安全。


# ファイルの削除

#### コマンド
```markdown
rm [オプション] <ファイル名1> <ファイル名2>....
```

#### 具体例
```
rm README.md
```

#### 概要
rmコマンドを実行すると、<mark>完全削除される</mark>。
ゴミ箱とかにはいかないので注意。

#### オプション

- ディレクトリも併せて削除する
	```
	rm -r dir
	```
	![[Pasted image 20260707130839.png]]
	※オプション無しでディレクトリを削除しようとした場合エラーとなる。

- ファイルを削除する際に警告文を表示しない
	`-r`と同時に使うことが多いが、削除してはいけないファイルまで削除してしまう可能性があるため注意。
	```
	rm -f file
	```

- ファイルの削除前に確認
	```
	rm -i file
	```

# ファイルの移動・ファイル名の変更

#### コマンド
```markdown
mv [オプション] <移動元> <移動先>
```

#### 具体例
移動先にファイルを指定するか、ディレクトリを指定するかで挙動が違う。

- 移動先がファイルの場合
	ファイル名が変更される
	```
	mv file file1
	```

- 移動先がディレクトリの場合
	ファイルの移動ができる
	```
	mv file /dir1`
	```

#### オプション
- 上書きする前に確認する
```
mv -i file file1
```

既にfile1が存在する場合、<mark>fileがfile1を上書きしてしまう</mark>。
それを防ぎたい場合は`-i`を使う。

# ファイル・ディレクトリのコピー

#### コマンド
```markdown
cp [オプション] <コピー元> <コピー先>
```

#### 具体例
- ファイルをコピー
```markdown
cp file new_file
```
※`file`をコピーした新しい`new_file`ができる。

- ファイルをディレクトリ内にコピー
```markdown
cp file dir/
```
※`file`を`dir/`内にコピーする。

⚠️コピー先のファイルが既にあると上書きされてしまうため注意。

#### オプション
- 上書きする前に確認する
```markdown
cp -i file new_file
```
`new_file`が存在するときに警告文を出す。

- ディレクトリをコピーする
```
cp -r dir dir new_dir
```

# リンクを張る

### リンクとは
Linuxではファイルに別名を付けることができ、これをリンクという。
<mark>Windowsでいうショートカット</mark>のようなもの。

**ハードリンク**

**シンボリックリンク**

### lnコマンド
#### コマンド
- ハードリンクを作成
```markdown
ln [オプション] <リンク元ファイル> <リンク名>
```

#### 具体例
- ハードリンク
```markdown
ls file1 file2
```
※オプションを何もつけないとハードリンクとなる。

- シンボリックリンク
```
ls -s file1 file2
```
※`-s`をつけるとシンボリックリンクとなる。

#### どういうときに使うか？
- 長いパス名の省略
- 複数バージョンの共存させ最新を区別したいとき
...etc

# ファイルの検索

#### コマンド
```
find <検索開始ディレクトリ> <検索条件> <アクション>
```

#### 具体例
```markdown
find . -name README.md -print
```
![[Pasted image 20260707134559.png]]

`-print`
パスを表示するアクション。
省略可能。

#### 検索条件
- ファイル名を指定してファイルを検索(大文字小文字を区別)
	`-name`
	ワイルドカードが使える。\*を使って指定するときは"で囲う
	
	```
	find . -name "*.html" -print
	```

💡クォーテーションで囲まないとP`パス名展開`となってしまい、エラーとなる。
- 【例】
	カレントディレクトリに、
	- `index.html`
	- `home.html`
	があるとする。
	このとき、
	`find . -name *.html -print`
	としてしまうと、内部的に
	`find . -name index.html home.html -print`
	となり、パス名を複数指定したことになり、エラーとなる。


- ファイル名を指定してファイルを検索(大文字小文字を区別しない)
	`-iname`
	```
	find . -iname readme.md
	```

- ファイルの種類で検索
	`-type f` : 通常ファイル
	`-type l`：シンボリックリンク
	`-type d`：ディレクトリ
	```
	find . -type f -print
	```
	💡`-name`など、他の検索条件と一緒に使うことが多い。

- 複数の検索条件を指定。なお、`-a`は省略可能
	```
	find . -type d -a -name images -print
	```
	※名前が`images`のディレクトリを探すという検索条件
